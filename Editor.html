The editor application is a Windows destkop application that allows editting of source files as well as debugging with the interpreter.
It is intended to be ran from Visual Studio for debugging purposes.

Under the edit menu item there's a configuration item (Options) that enables configuration of:

  • The default location for source files.
  • The default file extension for source files.
  • Enabling debug mode.

Setting the default source file location will cause the open/save/as options to initally start in this folder.

Setting the default file extension sets the filename of a newly created file to "new.<fileextension>".

Enabling debug mode is intended for the test case where the editor is being debugged from Visual Studio.
With debug mode enabled, exceptions are not caught in a try/catch block causing Visual Studio to break
execution at the location where the exception occurs, allowing for a better debugging experience.

The editor is a simple text editor with line numbers showing.
The text in the editor can be saved and new text can be loaded from files.
Text can be searched through with the standard find (Ctrl-F) text searcher.
If multiple ocurrances existed F3 will search for the next matching text part.

At the bottom of the editor there are 4 tabs containing the below information if the file was executed through debug/debug.

  • Output -> Shows the output of the interpreter.
  • Ast Tree -> Shows the AST tree created by the parser.
  • Scope Tree -> Shows the tree of scopes detected/created by the interpreter.
  • Lexer output -> Shows the list of tokens that were created by the lexer.
